# 1 拾取功能

## 方案一：颜色编码拾取

**实现步骤：**

1. 为模型分配唯一ID
2. 基于ID为模型填充对应颜色，采用离屏渲染出颜色附件
3. 读取点击位置的颜色附件颜色，根据颜色推导出模型ID

**扩展：**

- 如果是拾取图元，可以利用图元ID（`gl_PrimitiveID`）内置变量
- 根据模型数量和图元数量，合理选择颜色附件的格式

## 方案二：射线相交拾取

通过计算鼠标射线与场景中物体的相交点来实现拾取功能。

# 2 文本渲染

## 有向距离场（SDF）文本渲染方案

SDF 纹理的每个纹素存储着其到轮廓的有向最短距离：

- **正值**：表示像素位于轮廓内侧
- **负值**：表示位于外侧
- **零值**：精确对应轮廓边界

**优势：** 采用有向距离场渲染文字，可以减少缩放后出现的锯齿与模糊。

# 2.3 物体轮廓

## 方案一：基于模板缓冲

- **原理**：利用模板缓冲。
  1. 先绘制物体，将物体覆盖区域的模板值标记为 1；
  2. 略微放大物体，设置模板测试为"仅在模板值不等于 1 时绘制"；
  3. 绘制扩大的物体，多出的边缘部分即为轮廓。
- **特点**：效果非常稳定，轮廓是实心的几何色块。
- **适用场景**：游戏中的单位选中高亮、简单的描边效果。

## 方案二：基于法线与视线方向

- **原理**：利用**视线向量（View Direction）与法线（Normal）的点积**。
  - 公式：`edge = 1.0 - max(0.0, dot(V, N))`
  - 当视线与法线几乎垂直时（即物体边缘），该值趋近于 1，产生发光感。
- **特点**：计算开销极低，不需要额外 Draw Call 或后处理。但它生成的更像是"光晕"而非硬核线条。
- **适用场景**：物体边缘光（Rim Light）、能量场、幽灵透明效果。

## 方案三：基于深度/法线变化

- **原理**：一种**后处理（Post-processing）**技术。
  1. 获取全屏深度图和法线图；
  2. 使用卷积核（如 Sobel 算子）检测像素间的深度突变或法线朝向突变；
  3. 差异超过阈值的像素被判定为边缘。
- **特点**：能精准捕捉物体相交线和内部转折线（如内角线）。
- **适用场景**：标准的卡通渲染、复杂场景的描边。

## 方案四：JFA 距离场 (Jump Flood Algorithm / SDF)

- **原理**：基于**有向距离场（SDF）**。
  1. 预计算出有向距离场；
  2. 根据距离值生成边缘。
- **特点**：抗锯齿效果极佳，支持非常粗且平滑的轮廓，且能轻松实现外发光、渐变描边。
- **适用场景**：高品质 UI 描边、对性能要求较高的精致外轮廓效果。

**对比总结：**

- **性能消耗**：点积 < 模板法 < 深度法 < JFA
- **线条质量**：JFA > 深度法 > 模板法 > 点积

# 2.5 模型重复渲染

- **实例化渲染（Instanced Rendering）**：使用单个 Draw Call 渲染多个相同模型的不同实例
- **间接绘制（Indirect Drawing）**：通过 GPU 间接缓冲区控制绘制命令
- **批处理（Batching）**：将多个绘制操作合并以提高效率
