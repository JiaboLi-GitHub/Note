# 1 红黑树

## 1.1 二叉搜索树

二叉搜索树具备以下特性：

- 左子树上所有结点的值均小于或等于它的根结点的值。
- 右子树上所有结点的值均大于或等于它的根结点的值。
- 左、右子树也分别为二叉排序树。

二叉搜索树容易退化为链表。

## 1.2 平衡二叉树

在二叉搜索树的基础上添加以下特性：

- 每个结点的左右子树的高度之差的绝对值（平衡因子）最多为1。

平衡二叉树插入节点和删除节点时可能会导致不平衡，不平衡的场景主要有四种：

- LL型：单右旋转
- RR型：单左旋转
- LR型：左旋 + 右旋
- RL型：右旋 + 左旋

右旋（顺时针）：

1. 选择一个节点作为新根
2. 将新根的右子树作为旧根的左子树
3. 将旧根作为新根的右子树

平衡二叉树具有严格的平衡条件，所以需要花费大量时间在树的调整上，故平衡二叉树适用于频繁查询的场景，不适合频繁插入的场景。

红黑树在查询速率和平衡调整中寻找平衡，整体性能更好。

## 1.3 红黑树

### 1.3.1 特性

1. （颜色属性）：节点非黑即红。
2. （根属性）：根节点一定是黑色。
3. （叶子属性）：叶子节点（空节点）一定是黑色。
4. （红色属性）：每个红色节点的两个子节点一定是黑色。
5. （黑色属性）：从任一节点到其每个叶子的所有路径，都包含相同数目的黑色节点。

### 1.3.2 恢复平衡的操作

- 变色
- 左旋
- 右旋

# 2 哈希表（散列表）

哈希表会出现哈希冲突，可以使用以下方法解决：

## 2.1 开链法

每个哈希表的槽包含一个链表，所有哈希值相同的元素都存储在同一个链表中。

**unordered_map基于链表法。**

## 2.2 开放寻址法

### 2.2.1 线性探测

发生冲突时，线性探测下一个槽位，直到找到空槽。

### 2.2.2 二次探测

发生冲突时，以二次方的增量探测下一个槽位，直到找到空槽。

二次增量：hash(key) + 1^2，hash(key) + 2^2，hash(key) + 3^2

开发寻址法初始空间有限，当哈希表的**负载系数**超过阈值时，会触发**重新整理**。即创建更大的哈希表，将原哈希表中的所有元素重新哈希到新表中。
